# Release 1.1 — Ticket Workflow

## Что добавлено

### Кнопочный UX
- Главное меню через inline-кнопки.
- Клиентские действия: новая заявка, мои заявки, подтверждение заявки, подтверждение финальной цены, отмена.
- Мастерские действия: взять заявку, отправить AI-цену клиенту, вручную задать цену/срок, начать работу, закрыть заявку.
- Админская очередь: список активных заявок и просмотр карточки заявки.

Команды оставлены как fallback: `/start`, `/cancel`, `/status`, `/my_orders`.

### Lifecycle заявки
Новый основной workflow:

1. `draft`
2. `waiting_photos`
3. `ai_analysis`
4. `diagnosed`
5. `new`
6. `assigned`
7. `price_offered`
8. `client_approved`
9. `in_progress`
10. `done`
11. `cancelled`

Legacy-статусы `accepted` и `waiting_approval` сохранены для совместимости со старой БД.

### Назначение мастера
- Мастер нажимает `🙋 Взять заявку`.
- Заявка получает `master_id`.
- Клиент получает уведомление о назначении мастера.
- Мастер видит свои активные работы через кнопку `🔧 Мои работы`.

### Финальная цена и срок
- Мастер может отправить клиенту AI-оценку как финальную цену.
- Мастер может вручную задать цену и срок сообщением в формате:

```text
1500; 1 день
```

- Клиент подтверждает финальную цену кнопкой.
- До подтверждения цены мастер не может перевести заявку в работу.

### Админская очередь
- Админ видит активные заявки через кнопку `🧭 Очередь сервиса`.
- Очередь показывает заявки в статусах: `new`, `assigned`, `price_offered`, `client_approved`.
- Админ может открыть карточку заявки и выполнить мастерские действия.

## Миграция
Добавлена миграция:

```text
alembic/versions/0002_v11_ticket_workflow.py
```

Она расширяет PostgreSQL enum `ticketstatus` новыми значениями:

- `DIAGNOSED`
- `PRICE_OFFERED`
- `CLIENT_APPROVED`
- `ASSIGNED`

## Проверка перед запуском

```bash
docker compose up --build -d
docker compose logs -f bot
```

Smoke-сценарий:

1. Клиент нажимает `🛴 Новая заявка`.
2. Клиент отправляет описание, телефон, фото.
3. Клиент подтверждает предварительную заявку.
4. Мастер нажимает `🙋 Взять заявку`.
5. Мастер отправляет `💰 AI цена клиенту` или `✏️ Цена/срок`.
6. Клиент подтверждает цену.
7. Мастер нажимает `▶️ Начать работу`.
8. Мастер нажимает `🏁 Готово`.

## Ограничения
- Редактирование цены/срока пока требует одного текстового сообщения от мастера. Это осознанный компромисс: весь остальной workflow кнопочный.
- Админ пока не выбирает конкретного мастера из списка. Назначение работает через `Взять заявку` от имени мастера/админа.
- Нет календаря с реальной занятостью мастеров; используется существующий простой слот.
